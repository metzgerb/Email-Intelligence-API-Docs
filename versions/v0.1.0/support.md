## Support & Issues

If you encounter problems, have questions, or want to request new features for the Email Intelligence API, please use [GitHub Issues](https://github.com/metzgerb/Email-Intelligence-API-Docs/issues) in the public repository. This helps track requests, provide faster responses, and share updates with all users.

### How to Submit an Issue

1. Go to the Issues page of the public repository.
2. Click **New issue**.
3. Choose the appropriate template or provide the following details:
   - **Type:** Bug, Feature Request, Question, or Documentation
   - **Endpoint:** Which API endpoint your request or issue relates to (e.g., `/email/validate`)
   - **Description:** Clear explanation of the issue or feature request
   - **Request & Response:** Include example request payload and response (if applicable)
   - **Environment:** Optional — API version, programming language, or tool used
4. Submit the issue. It will be reviewed and addressed as quickly as possible.

### Response Times

- **Bugs affecting functionality:** Usually within 24–48 hours
- **Feature requests or general questions:** Within a few business days

### Notes

- Avoid creating duplicate issues by checking existing ones first.
- For feature requests, be as specific as possible — this helps prioritize effectively.
